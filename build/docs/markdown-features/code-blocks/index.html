Блоки кода

Блоки кода в документации супер мощные 💪.

Заголовок блока с кодом​

Вы можете добавить заголовок к блоку с кодом, добавив ключ title после языка (поставьте между ними пробел).

```jsx title="/src/components/HelloCodeTitle.js"
function HelloCodeTitle(props) {
  return <h1>Привет, {props.name}</h1>;
}
```

Подсветка синтаксиса​

Блоки с кодом — это текстовые блоки, обернутые строками из 3 обратных кавычек. Пройдите по этой ссылке для ознакомления со спецификациями MDX.

```js
console.log('Каждый репозиторий должен иметь своего маскота.');
```

Укажите язык программирования на котором написан ваш код, и Docusaurus автоматически подсветит синтаксис с помощью Prism React Renderer.

http://localhost:3000
console.log('Каждый репозиторий должен иметь своего маскота.');

Тема редактора в блоке кода​

По умолчанию, в качестве темы подсветки синтаксиса в Prism, мы используем Palenight. Вы можете изменить ее на другую тему, указав ее в поле themeConfig.prism.theme в docusaurus.config.js.

Например, если вы предпочитаете использовать тему подсветки dracula:

docusaurus.config.js
module.exports = {
  themeConfig: {
    prism: {
      theme: require('prism-react-renderer/themes/dracula'),
    },
  },
};

Поскольку тема Prism — это просто JS объект, вы также можете написать свою собственную тему, если вас не устраивает стандартная. Docusaurus усевершенствовал темы github и vsDark, чтобы обеспечить более богатую подсветку, а вы можете посмотреть наши реализации этих тем в светлых и темных тонах.

Доступные языки​

По умолчанию Docusaurus поставляется с коллекцией стандартных языков.

caution

Некоторые популярные языки, такие как Java, C# или PHP, не включены по умолчанию.

Чтобы добавить подсветку синтаксиса для любого из других языков, поддерживаемых Prism, определите его в массиве дополнительных языков.

note

Каждый дополнительный язык должен быть валидным именем компонента Prism. Например, Prism сопоставит язык cs с csharp, но только prism-csharp.js существует как компонент, поэтому вам необходимо использовать additionalLanguages: ['csharp']. Вы можете просмотреть node_modules/prismjs/components, чтобы найти все доступные компоненты (языки).

Например, если вы хотите добавить подсветку для языка PowerShell:

docusaurus.config.js
module.exports = {
  // ...
  themeConfig: {
    prism: {
      additionalLanguages: ['powershell'],
    },
    // ...
  },
};

После добавления additionalLanguages перезапустите Docusaurus.

Если вы хотите добавить подсветку языков, еще не поддерживаемых Prism, вы можете подключить prism-include-languages:

npm run swizzle @docusaurus/theme-classic prism-include-languages

Он создаст prism-include-languages.js в папке src/theme. И вы сможете добавить подсветку синтаксиса для собственных языков, отредактировав prism-include-languages.js:

src/theme/prism-include-languages.js
const prismIncludeLanguages = (Prism) => {
  // ...

  additionalLanguages.forEach((lang) => {
    require(`prismjs/components/prism-${lang}`);
  });

  require('/path/to/your/prism-language-definition');

  // ...
};

Обратитесь к описанию подсветки официальных языков Prism, когда будете писать собственную подсветку языка.

Подсветка линий​

Подсветка комментариями​

Вы можете использовать комментарии с highlight-next-line, highlight-start и highlight-end, чтобы выбрать, какие строки нужно выделить.

```js
function HighlightSomeText(highlight) {
  if (highlight) {
    // highlight-next-line
    return 'Этот текст подсвечен!';
  }

  return 'Нечего подсвечивать';
}

function HighlightMoreText(highlight) {
  // highlight-start
  if (highlight) {
    return 'Этот диапазон строк подсвечен!';
  }
  // highlight-end

  return 'Нечего подсвечивать';
}
```

Выделение строкой метаданных​

Чтобы выделить несколько строк, разделите номера строк запятыми или используйте синтаксис диапазона для выбора фрагмента строк. Эта функция использует библиотеку parse-number-range, и вы можете найти больше разного синтаксиса в описании проекта.

import React from 'react';

function MyComponent(props) {
  if (props.isBar) {
    return <div>Bar</div>;
  }

  return <div>Foo</div>;
}

export default MyComponent;

Ниже мы расскажем, как можно расширить систему магических комментариев для определения пользовательских директив и их функций. Магические комментарии будут парситься только в том случае, если отсутствует метастрока выделения.

Собственные магические комментарии​

// highlight-next-line и // highlight-start и т. д. называются "магическими комментариями", потому что они распрасены и удалены, и их цель - добавить метаданные к следующей строке или секции, которая обернута этими комментариями. Вы можете объявить пользовательские магические комментарии в настройках темы. Например, вы можете зарегистрировать другой волшебный комментарий, который добавляет code-block-error-line имя класса:

module.exports = {
  themeConfig: {
    prism: {
      magicComments: [
        // Не забудьте также расширить имя класса выделения по умолчанию!
        {
          className: 'theme-code-block-highlighted-line',
          line: 'highlight-next-line',
          block: {start: 'highlight-start', end: 'highlight-end'},
        },
        {
          className: 'code-block-error-line',
          line: 'This will error',
        },
      ],
    },
  },
};
.code-block-error-line {
  background-color: #ff000020;
  display: block;
  margin: 0 calc(-1 * var(--ifm-pre-padding));
  padding: 0 var(--ifm-pre-padding);
  border-left: 3px solid #ff000080;
}

Если вы используете числовые диапазоны в мета-строке (синтаксис {1,3-4}), Docusaurus применит к первой записи magicComments имя класса. По умолчанию это theme-code-block-highlighted-line, но если вы измените конфигурацию magicComments и используете другую запись в качестве первой, значение диапазона метастроки также изменится.

Вы можете отключить выделение строк комментариями по умолчанию с помощью magicComments: []. Если нет конфигурации магических комментариев, но Docusaurus обнаруживает блок кода, содержащий диапазон строк, он выдаст ошибку, потому что не будет имени класса для применения — в конце концов, имя класса выделения - это просто запись магического комментария.

Каждая запись магического комментария будет содержать три ключа: className (обязательно), line, который применяется непосредственно к следующей строке, или block (содержащий start и end), которые применяются ко всему блоку, заключенному в два комментария.

Использование CSS для выбора класса уже может многое сделать, но вы можете раскрыть весь потенциал этой функции с помощью swizzling.

npm run swizzle @docusaurus/theme-classic CodeBlock/Line

Компонент Line получит список имен классов, на основе которых можно условно отображать различную верстку.

Интерактивный редактор кода​

(Работает на React Live)

Вы можете создать интерактивный редактор кода с помощью плагина @docusaurus/theme-live-codeblock. Сначала добавьте плагин в ваши зависимости.

npm install --save @docusaurus/theme-live-codeblock

Далее, подключите плагин в docusaurus.config.js.

module.exports = {
  // ...
  themes: ['@docusaurus/theme-live-codeblock'],
  // ...
};

Чтобы использовать плагин, создайте блок кода с live, прикрепленным к метастроке языка.

```jsx live
function Clock(props) {
  const [date, setDate] = useState(new Date());
  useEffect(() => {
    const timerID = setInterval(() => tick(), 1000);

    return function cleanup() {
      clearInterval(timerID);
    };
  });

  function tick() {
    setDate(new Date());
  }

  return (
    <div>
      <h2>It is {date.toLocaleTimeString()}.</h2>
    </div>
  );
}
```

Блок кода будет отображаться как интерактивный редактор. Изменения в коде отразятся на панели результатов в реальном времени.

http://localhost:3000

Императивный рендеринг (noInline)​

Параметр noInline следует использовать, чтобы избежать ошибок, когда ваш код охватывает несколько компонентов или переменных.

```jsx live noInline
const project = 'Docusaurus';

const Greeting = () => <p>Привет {project}!</p>;

render(<Greeting />);
```

В отличие от обычного интерактивного блока кода, при использовании noInline React Live не будет оборачивать ваш код встроенной функцией для его рендеринга. Вам нужно будет явно вызвать render() в конце вашего кода, чтобы отобразить вывод.

http://localhost:3000
const project = "Docusaurus";

const Greeting = () => (
  <p>Привет {project}!</p>
);

render(
  <Greeting />
);

Использование JSX в блоках кода​

Блок кода в Markdown всегда сохраняет свое содержимое в виде простого текста, то есть вы не можете сделать что-то вроде:

type EditUrlFunction = (params: {
  // Это не превращается в ссылку (по уважительной причине!)
  version: <a href="/docs/versioning">Version</a>;
  versionDocsDirPath: string;
  docPath: string;
  permalink: string;
  locale: string;
}) => string | undefined;

Если вы хотите внедрить HTML-разметку, такую как якорные ссылки или полужирный шрифт, вы можете использовать тег <pre>, тег <code> или компонент <CodeBlock>.

<pre>
  <b>Ввод: </b>1 2 3 4{'\n'}
  <b>Вывод: </b>"366300745"{'\n'}
</pre>
http://localhost:3000
Ввод: 1 2 3 4
Вывод: "366300745"
MDX нечувствителен к пробелам

MDX соответствует поведению JSX: символы разрыва строки, даже находящиеся внутри <pre>, превращаются в пробелы. Вы должны явно написать символ новой строки, чтобы он был выведен.

caution

Подсветка синтаксиса работает только с простыми строками. Docusaurus не будет пытаться парсить содержимое блока кода, с дочерними элементами JSX внутри.

Поддержка нескольких языков в блоках​

С помощью MDX вы можете легко создавать интерактивные компоненты в своей документации, например, для отображения кода на нескольких языках программирования и переключения между ними с помощью компонента вкладок.

Вместо реализации отдельного компонента для блоков кода многоязычной поддержки мы реализовали универсальный <Tabs> в классической теме, чтобы вы могли использовать его и в других сценариях, не связанных с кодом.

В следующем примере показано, как вы можете использовать многоязычные вкладки кода в своей документации. Обратите внимание, что пустые строки над и под каждым языковым блоком добавлены преднамеренно. Это текущее ограничение MDX: вы должны оставлять пустые строки вокруг синтаксиса Markdown, чтобы MDX парсер знал, что это синтаксис Markdown, а не JSX.

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
<TabItem value="js" label="JavaScript">

```js
function helloWorld() {
  console.log('Привет, мир!');
}
```

</TabItem>
<TabItem value="py" label="Python">

```py
def hello_world():
  print("Привет, мир!")
```

</TabItem>
<TabItem value="java" label="Java">

```java
class HelloWorld {
  public static void main(String args[]) {
    System.out.println("Привет, Мир");
  }
}
```

</TabItem>
</Tabs>

И вы получите следующее:

Docusaurus npm2yarn remark плагин​

Отображение команд CLI как в npm, так и в Yarn — очень распространенная потребность, например:

npm install @docusaurus/remark-plugin-npm2yarn

Docusaurus предоставляет такую утилиту из коробки, освобождая вас от необходимости каждый раз использовать компонент Tabs. Чтобы включить эту функцию, сначала установите пакет @docusaurus/remark-plugin-npm2yarn, как указано выше, а затем в docusaurus.config.js в плагины, для которых это необходимо (doc, blog, pages, т. д.), зарегистрируйте ее в опции remarkPlugins. (Подробнее о формате конфигурации см. в конфигурации плагина Docs)

docusaurus.config.js
module.exports = {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          remarkPlugins: [
            [require('@docusaurus/remark-plugin-npm2yarn'), {sync: true}],
          ],
        },
        pages: {
          remarkPlugins: [require('@docusaurus/remark-plugin-npm2yarn')],
        },
        blog: {
          // ...
        },
      },
    ],
  ],
};

А затем используйте его, добавив ключ npm2yarn в блок кода:

```bash npm2yarn
npm install @docusaurus/remark-plugin-npm2yarn
```

Использование параметра {sync: true} синхронизирует все вкладки. Поскольку выбор хранится в одном и том же пространстве имен npm2yarn, разные экземпляры подключаемого модуля npm2yarn также будут синхронизировать свои варианты.

Использование в JSX​

Вне Markdown вы можете использовать компонент @theme/CodeBlock, чтобы получить тот же результат.

import CodeBlock from '@theme/CodeBlock';

export default function MyReactPage() {
  return (
    <div>
      <CodeBlock
        language="jsx"
        title="/src/components/HelloCodeTitle.js"
        showLineNumbers>
        {`function HelloCodeTitle(props) {
  return <h1>Привет, {props.name}</h1>;
}`}
      </CodeBlock>
    </div>
  );
}
http://localhost:3000
/src/components/HelloCodeTitle.js
function HelloCodeTitle(props) {  return <h1>Привет, {props.name}</h1>;}

Доступные свойства: language, title и showLineNumbers, точно такие же, как в Markdown.

Хотя это и не рекомендуется, вы также можете передать свойство metastring, например metastring='{1-2} title="/src/components/HelloCodeTitle.js" showLineNumbers', это то, как блоки кода в Markdown обрабатываются под капотом. Однако мы рекомендуем вам использовать комментарии для выделения строк.

Как указывалось ранее, подсветка синтаксиса применяется только в том случае, если дочерним элементом является простая строка.
