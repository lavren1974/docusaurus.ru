Документация может использовать любые стандартные [возможности Markdown](../markdown-features/markdown-features-intro.mdx) и иметь несколько дополнительных функций Markdown для исключительно для страниц документации.

## Markdown front matter {#markdown-front-matter}

У Markdown страниц есть их собственный [Markdown front matter API](../../api/plugins/plugin-content-docs.md#markdown-front-matter).

## Ссылки на другие страницы документации {#referencing-other-documents}

Если вы хотите добавить ссылку на другую свою страницу документации, вы можете использовать относительный путь той страницы, на которую хотите сослаться.

Docusaurus преобразует путь к файлу в окончательный URL-адрес страницы (и удалит расширение `.md`).

Например, если вы находитесь в `folder/doc1.md` и хотите сделать ссылку на `folder/doc2.md`, `folder/subfolder/doc3.md` и `otherFolder/doc4.md`:

```md
Я ссылаюсь на [страницу](doc2.md).

Ссылка на другую [страницу в подпапке](subfolder/doc3.md).

[Относительная ссылка на страницу](../otherFolder/doc4.md) тоже работает.
```

:::tip

Использование относительных путей _файлов_ (с расширениями `.md`) вместо относительных ссылок _URL_ дает следующие преимущества:

- ссылки продолжат работать в интерфейсе GitHub
- вы можете настроить путь до страницы, не обновляя все ссылки
- при наличии разных версий страница будет ссылаться на другую страницу той же версии
- относительные ссылки могут сломаться, если вы обновите конфигурацию [`trailingSlash`](../../api/docusaurus.config.js.md#trailing-slash).

:::
