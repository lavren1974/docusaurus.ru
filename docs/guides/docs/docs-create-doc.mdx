# Создать документ

Создайте файл Markdown `greeting.md` и поместите его в `docs` каталог.

```bash
website # корневая директория вашего сайта
├── docs
│   └── greeting.md
├── src
│   └── pages
├── docusaurus.config.js
├── ...
```

```md
---
описание: Создайте страницу документа с богатым содержимым.
---

# Привет от Докузавра

Готовы ли вы создать сайт документации для вашего проекта с открытым исходным кодом?

## Заголовки

появится в оглавлении в правом верхнем углу

Чтобы ваши пользователи знали, о чем эта страница, не прокручивая ее вниз и даже не читая слишком много.

## По умолчанию в оглавлении будут только h2 и h3.

Вы можете настроить уровни заголовков TOC либо для каждого документа, либо в конфигурации темы.

Заголовки хорошо разнесены, поэтому иерархия понятна.

- списки помогут вам
- представить ключевые моменты
- что вы хотите, чтобы ваши пользователи запомнили
   - и вы можете вложить их
     - много раз

## Заголовки пользовательских идентификаторов {#custom-id}

С помощью синтаксиса `{#custom-id}` вы можете установить свой собственный идентификатор заголовка.
```

:::note примечание

Все файлы с префиксом подчёркивания (`_`) в папке `docs` рассматриваются как «фрагменты» страниц и по умолчанию игнорируются.

Подробнее об [импорте фрагментов страниц](../markdown-features/markdown-features-react.mdx#importing-markdown).

## Страничный front matter {#doc-front-matter}

[front matter](../markdown-features/markdown-features-intro.mdx#front-matter) используется для предоставления дополнительных метаданных для страницы. Front matter необязателен— Docusaurus сможет вывести все необходимые метаданные и без него. Но, например, описанные далее [теги страницы](#dog-tags) можно описать только с помощью front matter. Все возможные поля смотрите в [руководстве по API](../../api/plugins/plugin-content-docs.md#markdown-front-matter).

## Теги в страницах документации {#doc-tags}

При желании вы можете добавить теги на свои страницы документации, и добавить еще один блок с категориями в дополнение к [боковой панели документов](./sidebar/index.md). Теги перечисляются в верхней части страницы в виде списка меток:

```md "your-doc-page.md"
---
id: doc-with-tags
title: Страница с тегами
tags:
  - Демо
  - Начало работы
---
```

:::tip совет

Теги также можно перечислить в виде массива `tags: [Демо, Начало работы]`.

Узнайте больше обо всём [синтаксисе массивов Yaml](https://www.w3schools.io/file/yaml-arrays/).

:::

## Организация структуры папок {#organizing-folder-structure}

Расположение Markdown файлов в папке `docs` влияет на генерацию страниц Docusaurus. Однако большинство из них можно отделить от файловой структуры.

### Идентификатор (ID) документа {#document-id}

Каждый документ имеет уникальный `id`. По умолчанию `id` документа — это имя документа (без расширения) относительно корневого каталога документов.

Например, идентификатор `greeting.md` — это `greeting`, а идентификатор `guide/hello.md` — это `guide/hello`.

```bash
website # Корневая директория вашего сайта
└── docs
   ├── greeting.md
   └── guide
      └── hello.md
```

Однако **окончание** `id` может быть определено пользователем в параметрах страницы. Например, если содержимое `guide/hello.md` определено, как показано ниже, его окончательным значением `id` будет `guide/part1`.

```md
---
id: part1
---

Lorem ipsum
```

ID используется для ссылки на страницу документации при создании вручную боковых панелей или при использовании компонентов макета или хуков, связанных со страницами документации.

### URL-адреса страниц документации {#doc-urls}

По умолчанию URL-адрес документа — это путь к файлу относительно папки `docs`. Используйте параметр `slug`, чтобы изменить URL-адрес страницы.

Например, предположим, что структура вашего сайта выглядит так:

```bash
website # Корневая директория вашего сайта
└── docs
    └── guide
        └── hello.md
```

По умолчанию страница `hello.md` будет открываться по адресу `/docs/guide/hello`. Вы можете изменить ее URL-адрес на `/docs/bonjour`:

```md
---
slug: /bonjour
---

Lorem ipsum
```

`slug` будет добавлен к `routeBasePath`, который по умолчанию равен `/docs`. Смотрите руководство по [режиму "только документация"](#docs-only-mode), чтобы узнать, как удалить часть `/docs` из URL-адреса.

:::note примечание

Можно использовать:

- абсолютные пути: `slug: /mySlug`, `slug: /`...
- относительные пути: `slug: mySlug`, `slug: ./../mySlug`...

:::

Если вы хотите, чтобы страница была доступна в корне и имела адрес вида `https://docusaurus.io/docs/`, вы можете использовать параметр slug:

```md
---
id: my-home-doc
slug: /
---

Lorem ipsum
```

### Боковые панели {#sidebars}

При использовании [автоматически сгенерированных боковых панелей](./sidebar/autogenerated.md) файловая структура будет определять структуру боковой панели.

Наша рекомендация по организации файловой системы: сделайте так, чтобы ваша файловая система отражала структуру боковой панели (чтобы вам не нужно было писать файл `sidebars.js` от руки) и используйте параметр `slug` во front matter для настройки URL-адреса каждой страницы.
