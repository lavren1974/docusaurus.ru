# Метаданные в head

## Настройка метаданных {#customizing-head-metadata}

Docusaurus автоматически устанавливает полезные метаданные страницы в `<html>`, `<head>` и `<body>` за вас. Можно добавить дополнительные метаданные (или переопределить существующие) с помощью тега `<head>` в файлах Markdown:

```md title="markdown-features-head-metadata.mdx"
---
id: head-metadata
title: Head Metadata
---

<!-- highlight-start -->
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
<!-- highlight-end -->

# Head Metadata

My text
```

```mdx-code-block
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
```

Тег `<head>` был добавлен в текущую страницу в качестве демонстрации. Откройте инструменты разработчика в браузере и проверьте, как изменились метаданные этой страницы.

:::note примечание

Эта функциональность построена на основе компонента Docusaurus [`<Head>`](./../../docusaurus-core.md#head). Обратитесь к [react-helmet](https://github.com/nfl/react-helmet) для получения исчерпывающей документации.

:::

:::tip совет Для обычного SEO вам это не нужно

Контентные плагины (например, docs и blog) предоставляют параметры во front matter, такие как `description`, `keywords` и `image`, которые будут автоматически применяться как к `description` и `og:description`, тогда как при использовании тега `<head>` вам придется вручную объявить два тега метаданных.

:::

## Description страницы {#markdown-page-description}

Метаданные из description используются в большем количестве мест, чем метаданные из head. Например, [генератор индексной страницы категории](../docs/sidebar/items.md#generated-index-page) из плагина docs использует метаданные из description для создания карточек внутренних страниц.

По умолчанию description является первой строкой содержания, которая может быть преобразована в обычный текст страницы. Например, следующий файл...

```md
# Заголовок

Основной контент... Может содержать некоторые [ссылки](./file.md) или **выделение**.
```

...по умолчанию будет иметь description "Основное содержание... Может содержать некоторые ссылки или выделение". Однако **такой подход не особо функционален**. Если первая строка не подходит в качестве description, всегда можно явно указать description во front matter страницы:

```md
---
description: Это описание переопределит значение по умолчанию.
---

# Заголовок

Основной контент... Может содержать некоторые [ссылки](./file.md) или **выделение**.
```
